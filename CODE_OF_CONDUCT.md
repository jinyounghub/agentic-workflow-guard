# Code of Conduct

This project expects respectful, constructive participation.

## Standards

- Be kind and specific in feedback.
- Assume good intent while still being clear about security risk.
- Do not harass, threaten, or target maintainers or users.
- Do not publish private vulnerability details without permission.

## Enforcement

Maintainers may remove comments, close issues, or block participants who violate these standards. Security reports should follow `SECURITY.md`.
